# 🔧 Fix 404 "Not Found" Error on Render

## Problem
You're getting a "404 Not Found" error when accessing your Render deployment.

## Solution

I've created a `router.php` file that will fix this issue. Follow these steps:

### Step 1: Update Your Render Service

1. Go to your **Render Dashboard**
2. Click on your **Web Service**
3. Go to **"Settings"** tab
4. Find **"Start Command"**
5. Change it from:
   ```
   php -S 0.0.0.0:$PORT -t .
   ```
   To:
   ```
   php -S 0.0.0.0:$PORT router.php
   ```
6. Click **"Save Changes"**

### Step 2: Commit and Push the Router File

The `router.php` file has been created. You need to commit and push it:

```bash
git add router.php
git commit -m "Add router.php to fix 404 errors"
git push origin main
```

Render will automatically redeploy with the new file.

### Step 3: Wait for Redeployment

- Go to your **Web Service** → **"Logs"** tab
- Wait for deployment to complete
- Your site should now work!

---

## Why This Happens

PHP's built-in server (`php -S`) doesn't automatically serve `index.html` as the default document like Apache does. The `router.php` script:

1. ✅ Serves `index.html` when accessing the root URL
2. ✅ Handles all file requests correctly
3. ✅ Processes PHP files normally
4. ✅ Returns proper 404 errors for missing files

---

## Alternative: Manual Fix in Render Dashboard

1. Go to **Web Service** → **"Settings"**
2. Change **Start Command** to: `php -S 0.0.0.0:$PORT router.php`
